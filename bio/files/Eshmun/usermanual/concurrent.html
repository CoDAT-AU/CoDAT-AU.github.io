Concurrent Kripke Structure

A Concurrent Kripke Structure is a model for concurrent programs, it is made by a collection of Pair Kripke Structures(*).

To create a new a Concurrent Kripke Structure go to File Menu -> New -> Concurrent Kripke Structure (Ctrl + m in Short).

This will change main panel into a tabbed panel, the title of each tab is the name of the contained sub Kripke Structure and the process-pair inside that structure.

Managing Structures: You can do the following:

  • Add a new structure: right click on any tab title -> Add. A Dialog will appear asking for the name of the new structure (has to be unique), and the process pair inside the structure (also unique).
  • Edit a structure: right click on the its title -> Edit. A Dialog will appear asking for the new name and the process pair, same constraint apply.
  • Remove a structure: right click on its title -> Remove.
  • Process Indices: The indices can be any string excluding white spaces and commas. The labels of states that reflect a process related functionality must be suffixed with the appropriate index.

    • Manipulating a sub-structure: You can Manipulate each structure in the same manner as the single kripke structure. Clear, Undo, and Redo only affect the current open sub structure.

    Auto generating Isomorphic pairs: You can auto generate Isomorphic pairs from any single Kripke structure. Open the desired single Kripke structure, click on the Edit -> Generate Pairs, then enter the specific number of pairs.
    The generated pairs will contain all distinct combination of 2 processes between process 1 to n. n being the number of pairs entered.

    (*) Pair Kripke Structure: For a single Kripke structure to be a valid Pair structure, it must contain two processes only, each transition can make a move on one of the two processes (not both), labels ending with a process index ( could be an integer or a string) are considered to be process specific, labels not ending with such an index are considered to be shared variables.
